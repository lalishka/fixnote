# FixNote Documentation

Read these documents first when starting work on FixNote:

- [PROJECT_STATE.md](PROJECT_STATE.md) - current state, completed work, and known gaps.
- [SETUP.md](SETUP.md) - local environment and commands.
- [TESTING.md](TESTING.md) - what counts as verified.
- [RUNBOOK.md](RUNBOOK.md) - repeatable project procedures.
- [DECISIONS.md](DECISIONS.md) - important product and technical decisions.
- [NEXT.md](NEXT.md) - next work items and blockers.
- [GLOSSARY.md](GLOSSARY.md) - stable project terms.

The source of truth for product behavior is the code in `src/` plus the current state notes in `PROJECT_STATE.md`.
